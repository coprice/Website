Algorithm Analysis with OpenAI Gym Games

About: I created this project with partners (Sam Oh, Brandon Hills) for my Artificial Intelligence class. We calculated optimal solutions for three games provided in the OpenAI Gym Environments (Taxi, Frozen Lakes, N-Chain) and used these solutions as a baseline for testing the effectiveness of tabular Q-learning on these games. Taxi is a game where a taxi spawns randomly on a 5x5 grid, as well as a pick-up and drop-off point. The player must go to the pick-up location to get a passenger, and then drop them off at the drop-off point. Frozen Lake comes in two forms: a 4x4 grid and an 8x8 grid, where there are frozen locations and holes. The player must move from the top left corner to the bottom right corner, without falling into a hole (in which case they lose). However, each time a player moves in a direction, they have a one-third probability of going in that direction, and a one-third probability of going in the two adjacent directions each. N-Chain is a game where you can go forward one step or back to the beginning of the chain. Going backward gives a small reward, while going forward gives no reward unless you are at the end of the chain, in which you gain a large reward. However, there is a twenty percent chance that you take the opposite action you choose.

Above: visualization of some of our agents.

Implementation: The entire project was written using Python 2.7. We modelled Taxi as a search problem, and used A* search to find the optimal policy. Frozen Lakes and N-Chain are stochastic games, so we modelled them as Markov Decision Processes (MDPs) and used value iteration to create optimal policies for them. For all of the games, we created a tabular Q-learning agent. For Taxi and Frozen Lakes, we also created an "Incentivized" Q-Learning agent, which is given extra rewards for moving closer to a goal state as well as penalties for moving farther from a goal state. After tuning hyperparameters, we created test files for each of the three games to test the performance of our reinforcement learning algorithms with varying training iterations. Visualization: The games that we worked on did not come with a graphical display. So, I created a GUI for each of the games, which is demonstrated in the walkthrough video at the top of the page. The GUI can be run from the graphics.py file, which allows you to choose which game to display, which agent to use, how many games to simulate, and how many training iterations a Q-learning agent should take.
